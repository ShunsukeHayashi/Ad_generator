# システムアーキテクチャ

## 全体アーキテクチャ

「Ad Creative Template v1.0」は、Mastraフレームワークを活用したマイクロサービスアーキテクチャに基づいています。システムは以下の主要レイヤーで構成されています：

1. **プレゼンテーション層**：ユーザーインターフェースとAPI Gateway
2. **アプリケーション層**：ワークフローエンジンと各種マイクロサービス
3. **データ層**：データベース、キャッシュ、ストレージ
4. **インフラストラクチャ層**：Kubernetes、モニタリング、ロギング

![システムアーキテクチャ図](images/architecture_diagram.png)

## マイクロサービスアーキテクチャ

システムは以下の独立したマイクロサービスで構成されています：

1. **API Gateway**：すべてのクライアントリクエストのエントリーポイント
2. **ワークフローエンジン**：Mastraフレームワークを使用したワークフロー管理
3. **画像解析サービス**：入力画像の特徴抽出
4. **テンプレートサービス**：広告テンプレートの生成と管理
5. **プロンプト生成サービス**：画像生成用プロンプトの最適化
6. **画像生成サービス**：OpenAI GPT-Image-1を使用した画像生成
7. **認証・認可サービス**：ユーザー認証と権限管理

## 通信パターン

マイクロサービス間の通信には、以下のパターンを採用します：

1. **同期通信**：REST API（HTTP/JSON）
   - クライアント-API Gateway間
   - API Gateway-マイクロサービス間（短時間で完了する処理）

2. **非同期通信**：メッセージキュー（RabbitMQ/Kafka）
   - 長時間実行される処理（画像生成など）
   - イベント通知（ワークフロー状態変更など）

3. **サービスディスカバリ**：Kubernetes Service/DNS

## デプロイメントアーキテクチャ

システムは、Kubernetesクラスター上にデプロイされます：

1. **Webティア**：Web UIとAPI Gateway
2. **アプリケーションティア**：ワークフローエンジンと各種マイクロサービス
3. **データティア**：データベース、キャッシュ、ストレージ

各マイクロサービスは、独自のDockerコンテナとしてパッケージ化され、Kubernetesポッドとしてデプロイされます。

## Mastraフレームワーク統合

システムは、Mastraフレームワークの以下のコンポーネントを活用しています：

1. **Workflow**：広告生成プロセスのステップを定義
2. **Agent**：ワークフローを実行し、ツールを呼び出す
3. **Tools**：画像解析、テンプレート生成、プロンプト生成、画像生成などの機能を提供
4. **Memory**：ワークフロー状態とコンテキストを保持
5. **Integrations**：OpenAI API、データベース、ストレージなどの外部サービスと連携
